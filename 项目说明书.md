# Tourgo 项目说明书

> **文档作用**：说明项目是做什么的、有哪些界面、各界面负责什么、模块与数据之间「谁跟谁是一伙的」（依赖与边界）。  
> **实现细节与目录树**：见 [README.md](./README.md) 与 [ARCHITECTURE.md](./ARCHITECTURE.md)。  
> **版本与构建号**：见 [VERSIONS.md](./VERSIONS.md)。

## 1. 项目是干什么的

**Tourgo** 是一个基于 **Firebase** 的、在真实城市里玩的 Web 应用平台，当前仓库里 shipped 了 **两套前端**，共用 **同一套后端项目**（同一个 Firebase 项目里的 Auth、Firestore、Storage、Hosting、Cloud Functions）：

| 产品 | 路径入口 | 一句话 |
|------|-----------|--------|
| **Tourgo NeoUI**（主应用） | `/NeoUI/` | 城市徒步 / 观光式 **Hunt 挑战**：浏览与创建多打卡点路线、计时跑步、拍照打卡、积分、地图浏览、排行榜与个人页。 |
| **Tourgo Friends / Manhunt**（子应用） | `/game/` | 好友向 **抓人游戏**：匿名进房、头像锁定、扫码、捕获拍照确认、类微信聊天（含语音）、在线状态与好友码。 |

两套应用 **不共享前端 JS 模块**（子应用是单文件 SPA），但 **共享 Firebase Auth 的 uid**；Firestore 里主业务与 Friend 使用 **不同集合命名空间**，避免数据混写。

---

## 2. 界面总览：一共几块「界面宇宙」

从用户能打开的 URL / 画面来数，可以分成 **三大块**：

1. **主应用 NeoUI**：一个 Hash 路由 SPA，多数「界面」对应 `#/页面名`（以及带 id 的深度页）。  
2. **Friend 子应用**：单页内用 **底部 Dock 标签** 切换「Start / Ranks / Social / Me」，进入房间后还有 **房内 Dock** 与 **观战 Dock**。  
3. **附属静态页**：根目录跳转、`tools/` 下工具页等（不参与主 SPA 路由表）。

下面按块列全。

---

## 3. 主应用 NeoUI：界面清单与职责

**技术入口**：`public/NeoUI/app.html` 加载 `public/js/app.js`，路由由 `public/js/lib/router.js`（解析 hash）+ `public/js/lib/route-dispatch.js`（分发到各 `pages/*.js`）驱动。  
**全局壳**：`public/js/components/shell.js` 提供顶栏、`#app-main` 内容区、底部 **五键 Dock**（Hunts · Saved · 创建 · Rank · Profile/登录）。

### 3.1 底部 Dock 与路由的对应关系（「一伙」）

Dock 是主应用的 **主导航**，与 hash 一一对应（创建为 FAB 指向 `#/create`）：

| Dock 按钮 | Hash | 激活的页面 key | 说明 |
|-----------|------|----------------|------|
| Hunts | `#/` | `hunts` | 首页 Hunt 列表（`home` 等别名同页） |
| Saved | `#/favorited` | `favorited` | 收藏列表（需登录；访客会被引导登录） |
| 中间 + | `#/create` | `create` | 创建 Hunt 入口（进一步分流到列表/地图/选点器） |
| Rank | `#/leaderboard` | `leaderboard` | 排行榜 |
| Profile / Sign in | `#/profile` 或 `#/login` | `profile` / `login` | 已登录进个人页，未登录进登录页 |

**地图浏览** `#/map` 等 **不在 Dock 上**，通常从首页或其它入口链过去。

### 3.2 全部 Hash 界面（路由表）

`page` 为 hash 第一段（`#/page` 或 `#/page/id`）。下列与 `route-dispatch.js` 中 `switch` 一致。

| `#/…` | 页面模块 | 主要职责 |
|--------|-----------|----------|
| `#/`、`#/home`、`#/overview`、`#/list` | `pages/home.js` | Hunt 列表浏览、进入详情 |
| `#/map` | `pages/map.js` | 地图模式浏览（Leaflet / 曼哈顿区域等） |
| `#/login` | `pages/login.js` | Google 登录、reCAPTCHA；已登录会跳回后续流程 |
| `#/profile` | `pages/profile.js` | 个人资料、积分展示相关 |
| `#/favorited` | `pages/favorited.js` | 收藏的 Hunt |
| `#/leaderboard` | `pages/leaderboard.js` | 排行榜 |
| `#/admin` | `pages/admin.js` | 管理员门户（callable / 规则内允许的运营能力） |
| `#/create` | `pages/create-map.js` 或 `pages/create-picker.js` | `id === "classic"` 时走地图创建，否则走选点器；访客需先登录 |
| `#/create-list` | `pages/create.js` | 创建流程中的「列表式」一步 |
| `#/create-map` | `pages/create-map.js` | 地图式创建 |
| `#/challenge/:id` | `pages/challenge.js` | 某一 Hunt 的详情与开始挑战 |
| `#/run/:id` | `pages/run.js` | 正在进行的一次 Run（计时、打卡、上传）；访客不可 |
| `#/hunt-review/:id` | `pages/hunt-review.js` | 结束后的复盘 / 照片与评论等；访客不可 |
| 未知 `page` | — | 壳内错误提示页 |

**认证与访客**：未登录且不允许匿名浏览时，会先被带到 `#/login` 并保存返回 hash；部分页面（收藏、创建、run、review）对 **Guest 浏览** 会弹窗引导登录。`login`、`admin`、`profile` 在规则上有特殊放行逻辑（见 `route-dispatch.js`）。

### 3.3 主应用里「流程上的一伙」（用户动线）

这些界面在 **产品故事** 上常连在一起，改其中一页要考虑上下游：

```
首页/地图发现 Hunt → challenge（详情/开始）→ run（进行中）→ hunt-review（结束后）
                                    ↘ favorited（从列表收藏）
create / create-picker / create-map / create-list → 新 Hunt → 回到 home / challenge
login ↔ profile（积分、显示名与 users 服务）
leaderboard ← 依赖 attempts / users 等积分数据（经 services）
admin ← 独立运营能力，依赖 admin-portal 与 Functions
```

## 4. Friend 子应用 `/game/`：界面清单与职责

**技术入口**：`public/game/index.html` + `public/game/js/app.js`（单文件大模块）+ `public/game/hide-seek.css`。  
**与主应用关系**：仅通过 **Firebase SDK** 与 **Firestore/Storage 合约字段** 与主应用「弱关联」（例如展示名可与主应用 `users` 同步）；**禁止** `import` 主应用 `public/js/` 下任何模块。

### 4.1 大厅阶段：底部四标签（主 Dock）

| 标签 id | 名称 | 职责 |
|---------|------|------|
| `start` | Start | 创建房间 / 加入房间入口、匿名与 Friend Code 展示 |
| `ranks` | Ranks | 本地/跨局统计类展示（Hall of Friends） |
| `social` | Social | 好友、聊天列表、会话与语音等 |
| `me` | Me | 个人设置、头像、与 Tourgo 主应用同步选项等 |

界面顶部可有 **返回 Tourgo** chip，把用户带回主应用站点。

### 4.2 房间内：玩家底部四标签（房内 Dock）

| 标签 id | 名称 | 职责 |
|---------|------|------|
| `status` | Status | 猎手/跑者各自的游戏状态 UI（列表、倒计时、操作） |
| `team` | Team | 队伍与成员相关视图 |
| `qr` | Scan / QR | 扫码或展示 QR（依角色文案不同） |
| `about` | About | 房间说明/规则类 |

### 4.3 观战 / 已出局：底部三标签（观战 Dock）

| 标签 id | 名称 | 职责 |
|---------|------|------|
| `stats` | Stats | 出局后的统计与英雄区文案 |
| `all` | All chat | 全局聊天视角 |
| `exit` | Exit | 离开当前视图/房间相关流程 |

### 4.4 Friend 专属数据「一伙」

以下集合 **只给 Friend 用**，与主应用 `challenges` / `attempts` 等 **不是同一伙数据**（规则与索引也分开考虑）：

- `hideSeekRooms`（含子集合 pending / members / captures）  
- `hideSeekPresence`  
- `hideSeekFriendIndex`  
- `hideSeekChats`（含 messages）

Storage 上头像、语音等路径以 `hide-seek/…` 等为约定；头像校验可走 Cloud Functions（见 `functions/headshot-validator.js`）。

## 6. 代码结构：谁跟谁是一伙（依赖关系）

### 6.1 主应用 JS 分层（严格单向）

```
lib/ → services/ → components/ → pages/ → app.js
```

| 层级 | 目录 | 「一伙」里的约定 |
|------|------|------------------|
| **lib** | `public/js/lib/` | 路由、主题、状态、工具函数；**不**依赖 services/components/pages。 |
| **services** | `public/js/services/` | **唯一**宜直接调用 Firebase SDK 的业务封装；pages 不直接 `import firebase/firestore` 散写。 |
| **components** | `public/js/components/` | 壳、弹窗、Run 中社交块等；依赖 lib + services。 |
| **pages** | `public/js/pages/` | 每个界面导出 `render` / `cleanup`；依赖 lib、services、components。 |
| **入口** | `public/js/app.js` | 挂载壳、合并 hash/auth 路由刷新、Dock 与 visualViewport 等。 |

**路由一伙**：`router.js`（解析）← `route-events.js`（`requestRoute` 队列）→ `route-dispatch.js`（选择 page）→ 某 `pages/*.js` → 几乎总会通过 `shell.js` 写入主内容区。

**样式一伙**：`public/css/app.css` 按约定顺序 `@import` tokens → base → components → layout → pages → 主题 → motion；改 Dock 动画需记着 **每次路由会重挂 body 内大块 DOM**，Dock 不宜做「每次导航都播一次」的入场动画（见 ARCHITECTURE）。

### 6.2 子应用 JS/CSS「一伙」

- **只有一伙**：`game/js/app.js` + `game/hide-seek.css` + `game/index.html`。  
- 内部用「章节注释 + 多个 render/bind」组织，**不**拆成与主应用共享的 ES 包。

### 6.3 后端 Cloud Functions「一伙」

- `functions/index.js`：导出 HTTP / Callable 等。  
- `functions/headshot-validator.js`：Friend 头像相关校验逻辑（由 index 挂接）。  
- 主应用管理员相关 handler 若在仓库中单独文件，与 `services/admin-portal.js`、Firestore 规则 **一伙**维护。

### 6.4 Firebase 平台服务「一伙」

| 服务 | 主应用主要用 | Friend 主要用 |
|------|-------------|---------------|
| Auth | Google + reCAPTCHA、Guest | 匿名登录 |
| Firestore | challenges, attempts, users, favorites, reports… | hideSeek* |
| Storage | Hunt 打卡图等 | 头像、语音消息 |
| Functions | 管理员、校验等 | 头像校验等 |
